# Установка lenin-core

Гайд: от нуля до живого Ленина на твоей машине.

> Если что-то не так — `python3 scripts/doctor.py` подскажет конкретно.

## Предпосылки
- macOS / Linux, Claude Code, python3 ≥ 3.8.
- Для Lean-формализации (опц.): `lake` (Mathlib тянется автоматически).

## 1. Поставить плагин
```
/plugin marketplace add https://github.com/nlarryelvis2-max/lenin-plugins.git
/plugin install lenin-core@lenin
```
(маркетплейс публичный — авторизация не нужна)

## 2. Перезапустить Claude Code
Команды и хуки плагина подхватываются при старте. Закрой и открой сеанс.

## 3. Онбординг
```
/lenin setup
```
Спросит: имя владельца + тип профиля (psych/cfo/ops/builder/designer/athlete) +
куда развернуть ядро (по умолчанию `~/.claude/lenin-kernel`). Создаст конфиг и
развернёт скелет ядра (CLAUDE.md, MEMORY.md, hot/, library/).

## 4. Открыть ядро
```
cd ~/.claude/lenin-kernel && claude
```
Ленин живой. На старте видишь 14D-веса (пока пустые — наполнятся с работы),
идентичность, ритуал сессии. Атомы сигналят на каждый промпт.

## 5. Проверка
```
/lenin              # статус ядра
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/doctor.py
```

## Что у тебя, что у создателя
- **У тебя (поведение):** 14D-движок, атомы, гейты, Lean-канон, ритуал, шаблоны.
- **Не у тебя (личное создателя):** его люди, проекты, пациенты, память. У тебя
  своя память — с нуля, наполняешь собой.

## Lean (опционально)
Формализация (T1/T★/T2-T8) лежит в `lean/`. Чтобы собрать: `cd lean && lake build`
(Mathlib скачается). Без lake — формализация как референс, плагин работает и так.

## Частые проблемы
| Симптом | Fix |
|---|---|
| `/lenin` неизвестная команда | Перезапусти Claude Code (шаг 2) |
| Веса «пустые» на старте | Норма — наполни observations работой, weights оживут со следующих сессий |
| Хуки молчат | `doctor.py` → проверь что плагин в кэше |
| Хочу сменить профиль | `rm ~/.claude/lenin/config.json && /lenin setup` |
