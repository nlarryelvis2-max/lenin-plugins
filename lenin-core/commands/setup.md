Онбординг Ленина под ключ. setup.py интерактивный — спросит всё сам. Если юзер
хочет неинтерактивно, можно передать `--owner --profile --dir`.

Параметры (узнай через AskUserQuestion или `$ARGUMENTS`):
1. **Имя владельца** — как обращаться.
2. **Тип профиля** — `psych`/`cfo`/`ops`/`builder`/`designer`/`athlete` (язык Ленина).
3. (опц.) **Путь ядра** — по умолчанию `~/.claude/lenin-kernel`.

Что делает setup.py (одной командой):
- спрашивает **анкету холодного старта** (сфера / что важно / контекст — опц., кормит 14D);
- разворачивает ядро (CLAUDE.md, MEMORY.md, hot/, library/) в фиксированную папку;
- ставит **launchd** для ежедневного синка (если lenin-uplink установлен);
- запускает **device-flow привязку к платформе** (если token ещё моковый): покажет
  код → юзер подтверждает на lenin.nglain.com → токен в config.

Запуск:
```
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/setup.py" $ARGUMENTS
```

После — покажи юзеру: куда развёрнуто ядро, привязан ли синк, и подскажи
`cd <путь> && claude` чтобы Ленин ожил. Если привязка к платформе не завершилась
(платформа недоступна) — скажи: `/uplink register` позже.
