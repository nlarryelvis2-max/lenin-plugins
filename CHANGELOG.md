# Changelog

## Lenin Client 0.2.4 · Core 0.1.5 · Uplink 1.1.4

- Приватный cache открытых бинарных материалов закрыт mode `0700` и ограничен
  40 файлами и 256 МБ; текущий материал сохраняется всегда, старые удаляются.
- Документация прямо разделяет live project snapshot, ленивое чтение
  материалов, единственную разрешённую запись в командный чат и raw Uplink.

## Lenin Client 0.2.3 · Core 0.1.5 · Uplink 1.1.4

- Агент может лениво открыть по названию доступный файл, публичный артефакт или
  знание, не загружая всю библиотеку в каждый turn.
- Текстовые материалы возвращаются в Terminal, бинарные сохраняются с mode
  `0600` в приватный cache и открываются агентом через Read.
- Сервер повторно проверяет project grant и видимость каждого материала;
  чужие приватные материалы остаются недоступны.

## Lenin Client 0.2.2 · Core 0.1.5 · Uplink 1.1.4

- Добавлен skill `work-with-lenin-projects`: агент сам распознаёт проектную
  задачу и лениво поднимает живые документы, задачи, capabilities, участников
  и командный чат без обязательной slash-команды.
- По естественной явной просьбе пользователя агент может отправить готовый
  текст в командный чат; остальные project writes остаются запрещены.
- Материалы и сообщения команды явно трактуются как данные, а не инструкции.

## Lenin Client 0.2.1 · Core 0.1.5 · Uplink 1.1.4

- Добавлена явная команда `projects send <проект> -- <сообщение>` для
  публикации текста в командный чат от имени подключённого пользователя.
- Project token по-прежнему не даёт общего write-доступа: запрещены вложения,
  вызов агента и любые изменения задач, документов, знаний или настроек.
- Проект и командный чат остаются живой серверной проекцией без локальной копии.

## Lenin Client 0.2.0 · Core 0.1.5 · Uplink 1.1.4

- Добавлена команда `/lenin-client:projects` для живого read-only доступа к
  назначенным проектам из Claude Code.
- Проектный снимок включает канонические документы, участников, задачи,
  доступные материалы, capabilities и последние сообщения командного чата.
- Project token выдаётся отдельным одноразовым кодом, хранится в macOS
  Keychain и не смешивается с write-only Uplink token.
- Терминальный клиент не создаёт локальную копию проекта и повторно проверяет
  текущие grants на каждом запросе.

## Lenin Client 0.1.1 · Core 0.1.5 · Uplink 1.1.4

- Синхронизация выключена до явного подключения одноразовым кодом.
- Добавлена команда `/lenin-client:uplink disconnect` для отзыва доступа Mac.
- При недоступной платформе локальная отправка останавливается, а token
  сохраняется для повторной попытки серверного отзыва.
- Пользовательские подсказки синхронизированы с постоянным разделом платформы
  **Установить Ленина**.

## Lenin Client 0.1.0 · Core 0.1.5 · Uplink 1.1.3

- Добавлен единый публичный пакет `lenin-client` без дублирования core/uplink.
- Регистрация согласована с приватной платформой через одноразовый setup-код.
- Token сохраняется локально с mode `0600` и не выводится.
- После регистрации отправляется безопасный heartbeat без сырых сессий.
- Heartbeat сообщает структурированные версии Client/Core/Uplink.
- Сохранены hardening-правки путей, атомарной записи, log rotation и защита от
  исчезновения активного session-файла между scan и read.
- Сохранена совместимость с отдельными компонентами `lenin-core` и
  `lenin-uplink`.
